# Blockers

Last reviewed: 2026-05-26

This file documents why each market is either implemented, guarded, or stubbed. The project rule is: use official APIs, SDKs, documented endpoints, or protocol contracts only. Do not scrape private data, bypass authentication, or enable live trading/copy trading by default.

## Summary

- Polymarket, Kalshi, and Manifold are implemented adapters today.
- All other markets are present in config and GUI through graceful stub adapters.
- Live trading remains disabled by default for every market.
- Credentials must stay in `.env`, shell environment variables, OS keychain tooling, or external credential files. They must not be stored in `data/config.json`.

## Implementation Rules For Clearing A Blocker

1. Add or update an adapter using official/documented APIs only.
2. Add offline fixture tests for every endpoint parser and order/dry-run path.
3. Keep paper/dry-run mode as default.
4. Keep live trading disabled unless the user explicitly enables it in local config.
5. Never commit credentials, cookies, tokens, API keys, private URLs, IPs, or user data.
6. Update `README.md`, `data/config.example.json`, and this file when a blocker changes.
